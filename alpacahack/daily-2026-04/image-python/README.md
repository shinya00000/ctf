# image-python writeup
<https://alpacahack.com/daily/challenges/image-python>
参考：<https://www.tohoho-web.com/wwwgif.htm>

## 1. 概要
本問題はPythonの `exec()` を用いたJail問題です．入力したデータは `file` コマンドによって画像ファイルであるか判定されます．画像ファイルとしての判定を突破しつつPythonコードとして有効なPolyglotを作成することで任意のコードを実行しフラグを取得します．

## 2. 問題の分析

### ファイル構成

* Dockerfile
* compose.yaml
* jail.py
* flag.txt

### ソースコードの確認
jail.pyではユーザー入力を16進数のバイト列として受け取ります．そのバイト列を `file --mime-type -b -` コマンドに渡しMIMEタイプを取得します．判定結果の文字列が `image/` から始まらない場合は不正な入力としてプログラムが終了します．このチェックを通過すると `exec(img)` によって入力データがそのままPythonスクリプトとして実行されます．

## 3. 脆弱性と攻略方針

### 脆弱性
ユーザーの入力データがサニタイズされずに直接 `exec()` に渡される点において脆弱性が存在します．事前の `file` コマンドによるチェックはファイル先頭のシグネチャなどに基づいた判定のみを行なっています．このチェックでは入力全体が有効な画像データであることを厳密に検証していません．

### 攻略方針：
画像ファイルとして認識されつつPythonコードとしても成立するデータ（Polyglot）を作成します．テキストベースのシグネチャを持つGIFフォーマットを利用します．GIFの先頭シグネチャである `GIF89a` をPythonの変数名として扱いダミーの数値を代入すると構文エラーを回避できます．この手法を用いてフラグを読み込むPythonスクリプトを記述します．最後に全体を16進数に変換して送信します．
