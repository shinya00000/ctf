# 2026-04 strange-filename Writeup
<https://alpacahack.com/daily/challenges/strange-filename?month=2026-04>

## 1. 概要
提供されたソースコードと設定ファイルからフラグが書かれた特殊な名前のファイルを読み込む問題です．コマンドの仕様を正しく理解し，特殊なファイル名を適切に指定して読み込む方法が問われます．

## 2. 問題の分析

### ファイル構成
配布されたアーカイブからはコンテナを構築してサーバーを立ち上げるための設定ファイル群とソースコードが確認できます．対象となる主なファイルは`server.py`と`Dockerfile`です．

### ソースコードの確認
`Dockerfile`を確認すると，`COPY flag.txt -`という記述によってフラグファイルがコンテナ内に配置されています．つまり，コンテナ内でのフラグのファイル名は`-`（ハイフン）になります．
また，サーバーの処理を行う`server.py`では，`subprocess.run(["cat", input("$ cat ")])`が実行されています．これにより，ユーザーが入力した文字列がそのまま`cat`コマンドの引数として渡される仕組みになっています．

## 3. 脆弱性と攻略方針

### 脆弱性
フラグを読み出すために`cat`コマンドの引数として単に`-`を入力すると問題が生じます．Linux環境の`cat`コマンドにおいて，単独の`-`は「標準入力（stdin）から読み込む」という意味として解釈される仕様があります．そのため，ファイル名としての`-`ではなく標準入力からの入力待ち状態となってしまい，ファイルの中身を表示することができません．

### 攻略方針：
`cat`コマンドに標準入力として解釈させるのではなく，カレントディレクトリに存在する`-`という名前のファイルとして明示的に認識させる必要があります．これを解決するためには，単一のハイフンではなく，相対パスを使ってファイルの位置を明確に指定して入力します．

## 4. エクスプロイトコード
プロンプトに対して，以下の文字列を入力します．

./-

これにより内部で`cat ./-`が実行されるため，標準入力として解釈されることを回避し，フラグが含まれたファイルの中身を正常に出力することができます．
