# unrevealed-txt writeup

## 1. 概要
本問題はDNSのゾーン転送（AXFR）の設定不備を利用して，隠されたTXTレコードからフラグを取得する問題です．

## 2. 問題の分析

### ファイル構成
Docker Composeを用いてDNSサーバー（BIND9）とPythonベースのアプリケーションサーバーが構築されています．

### ソースコードの確認
`app/server.py`ではユーザーの入力を受け取り，`shlex.split`で分割した後に`dig @dns`の引数として追加して実行しています．また`dns/named.conf`を確認すると`allow-transfer { any; };`と設定されており，任意のクライアントからゾーン転送を許可する状態となっています．さらに`dns/db.alpaca.internal`ではフラグの入ったTXTレコードのサブドメイン名がランダムな32文字の文字列に置換される仕組みになっています．

## 3. 脆弱性と攻略方針

### 脆弱性
DNSサーバーの設定において`allow-transfer { any; };`が指定されているため，誰でもドメイン内の全レコードを一括で取得できる脆弱性が存在します．

### 攻略方針：
フラグの入ったサブドメイン名はランダム化されているため直接推測することは困難です．しかしゾーン転送を利用することで対象ドメインの全レコードを抽出できます．アプリケーションサーバーの入力プロンプトに対してドメイン名とゾーン転送を要求するクエリタイプを指定することでフラグを取得します．

## 4. エクスプロイトコード
アプリケーションの入力プロンプトに対して以下の文字列を入力します．

```bash
alpaca.internal axfr
```

これにより背後で`dig @dns alpaca.internal axfr`が実行され，全レコードが転送されます．出力結果のTXTレコードからフラグが得られます．
