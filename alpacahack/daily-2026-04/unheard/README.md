# unheard writeup
<https://alpacahack.com/daily/challenges/unheard>

## 1. 概要
本問題は制限されたPython環境（PyJail）からフラグを読み取る問題です．

## 2. 問題の分析

### ファイル構成
* `Dockerfile`
* `compose.yaml`
* `jail.py`

### ソースコードの確認
`jail.py` を確認すると以下のことがわかります．
* 監査フックが登録される前に `os.open("flag.txt", os.O_RDONLY)` が実行されファイルディスクリプタが確保されている．
* その後 `os` は `del` によって削除されている．
* 入力したコードは `{"__builtins__": __builtins__, "sys": sys}` という名前空間において実行される．

## 3. 脆弱性と攻略方針

### 脆弱性
名前空間の制限および監査フックによる制限が存在しますが，以下の2点が脆弱となります．
1. `sys` モジュールが提供されているため `sys.modules['os']` を通じて `os` モジュールに再アクセス可能であること．
2. Pythonの監査イベントは `os.open` には設定されているが，すでに開かれたファイルに対する `os.read` 操作には設定されていないこと．

### 攻略方針：
ヒント2で監査フックのバイパスが示唆されていますがこれは罠（レッドへリング）です．
実際には監査フックを回避する必要はなく，すでに開かれているファイルディスクリプタ（通常プロセスで最初に開かれるファイルは `3`）をそのまま読み取ります．
`sys.modules['os'].read(3, 100)` を実行することでフックに検知されることなくフラグを取得できます．

## 4. エクスプロイトコード
```bash
print(sys.modules['os'].read(3, 100).decode())
```
