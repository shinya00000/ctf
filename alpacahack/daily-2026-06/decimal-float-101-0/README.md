# Decimal float 101.0 writeup
<https://alpacahack.com/daily/challenges/decimal-float-101-0>

## 1. 概要

本問題はC言語のIEEE 754-2008規格で定義された `_Decimal64` 型（十進浮動小数点数）を扱う，PwnおよびMiscジャンルの問題である．数値として `0.1` に等しくかつメモリ上のビット表現が異なる値を標準入力から与えることでフラグを獲得できる．

## 2. 問題の分析

### ファイル構成

配布されたファイルはソースコードと実行ファイルのみであると推測される．またサーバー上にはフラグが記載された `flag.txt` が配置されている．

### ソースコードの確認

配布されたC言語のソースコードを確認したところ，以下の処理が行われていることが分かる．

1. `_Decimal64` 型の変数 `x_dd` に `0.1dd` を代入しそのメモリ上のビット列を16進数文字列（`31A0000000000001`）として出力する．
2. 標準入力から16進数を受け取り8バイト整数 `y_ul` に格納した後，`_Decimal64` 型の変数 `y_dd` にそのままメモリコピーする．
3. `x_dd != y_dd` ならば失敗する．
4. `memcmp(&x_dd, &y_dd, 8) == 0` ならば失敗する．
5. 両方の条件を通過した場合，`win()` 関数が呼び出され，`flag.txt` の内容が出力される．

## 3. 脆弱性と攻略方針

### 脆弱性

通常の二進浮動小数点数とは異なり，`_Decimal64` 型には同じ数値に対して複数のビット表現を許容する「コホート」という概念が存在する．比較演算子 `==` は数値としての等価性を判定する一方，`memcmp` はメモリの生バイト列を直接比較するため，この2つの判定基準に乖離が生じる点が本問題の脆弱性である．

### 攻略方針：

数学的に，$1 \times 10^{-1}$ と $10 \times 10^{-2}$ は等しい数値である．ヒントとして提示される `31A0000000000001` は仮数 $1$ および指数 $-1$ で構成されているため，仮数を $10$ にして指数を $-2$ に変更した，別のビット表現を作成すればよい．

`_Decimal64`（BID方式）のビット構成は仮数が小さい場合以下のようになる．

* 符号ビット（1ビット）: 正のため `0`
* バイアス付き指数部（10ビット）: バイアス値 $398$ に指数を加算
* 仮数部（53ビット）: 整数としての仮数値

仮数 $10$ かつ指数 $-2$ の場合バイアス付き指数は $-2 + 398 = 396$（16進数で `0x18C`）となる．これを53ビット左シフトして仮数 $10$（`0xA`）との論理和をとることで別の表現である `318000000000000A` を導出できる．この値を標準入力に与えることで両方のチェックを突破できる．

## 4. エクスプロイトコード

以下は条件を満たす入力値を算出しサーバーに送信してフラグを獲得するPythonソルバである．

```python
from pwn import *

def solve():
    # ローカルプロセスまたはリモートサーバーに接続
    # p = remote("example.com", 1337)
    p = process("./chall")

    # ヒント文字列を受信
    p.recvuntil(b"hint: ")
    hint = p.recvline().strip().decode()
    log.info(f"Hint received: {hint}")

    # コホートを利用して 10 * 10^-2 のビット列を生成
    significand = 10
    exponent = -2
    biased_exponent = exponent + 398
    payload_int = (biased_exponent << 53) | significand
    payload = f"{payload_int:016X}"

    log.info(f"Sending payload: {payload}")
    p.sendline(payload.encode())

    # フラグの出力を受信
    p.recvuntil(b"Here is the FLAG: ")
    flag = p.recvline().strip().decode()
    log.success(f"FLAG: {flag}")

    p.close()

if __name__ == "__main__":
    solve()

```
