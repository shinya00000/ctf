# func-array writeup
<https://alpacahack.com/daily/challenges/func-array>

## 1. 概要

本問題はOut-of-Bounds（境界外読み込み）の脆弱性を利用してスタック上のリターンアドレスを関数ポインタとして呼び出しシェルを奪取するPwn問題です．

## 2. 問題の分析

### ファイル構成

Dockerfile，compose.yaml，chal.cのソースコードおよびフラグファイル（flag.txt）が提供されています．

### ソースコードの確認

chal.cのvuln関数においてalpaca_functions配列に関数ポインタを格納し，入力されたインデックス番号を用いて関数を呼び出す処理が存在します．
またmain関数においてvuln関数の呼び出し直後にシェルを起動するwin関数が配置されています．

## 3. 脆弱性と攻略方針

### 脆弱性

vuln関数においてscanfで入力されたchoiceに対する境界チェックが存在しません．
3以上の数値を入力するとスタック上の配列範囲外のメモリを読み込みそれを関数ポインタとして実行してしまう脆弱性（Out-of-Bounds Read & Execute）があります．

### 攻略方針：

関数呼び出しの際にスタックには呼び出し元に戻るためのリターンアドレスが積まれます．
本プログラムではmain関数におけるvuln関数の呼び出しの次の命令がwin関数となっているためスタック上のリターンアドレスはwin関数の呼び出し命令を指しています．
このリターンアドレスが配置されているスタック上の位置を特定し，choiceとして入力することでwin関数を実行させます．
リモートサーバーに接続してインデックスとして5を入力するとwin関数が実行されシェルを奪取できます．

## 4. エクスプロイトコード

ターミナルからnetcatを用いてリモートサーバーに接続し，インデックスとして5を入力してシェルを獲得します．その後フラグファイルを読み込みます．

```bash
$ nc 34.170.146.252 38015
Select an alpaca function index: 5
You Win! 🦙🦙🦙🦙🦙
cat flag.txt
Alpaca{...}

```
