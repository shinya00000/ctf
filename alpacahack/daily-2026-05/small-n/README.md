# small-n writeup

## 1. 概要
本問題はRSA暗号において素因数の一つが32ビットと極端に小さいという脆弱性を突き，ポラード・ロー素因数分解法を用いることでフラグを復元する問題です．

## 2. 問題の分析

### ファイル構成
* `output.txt`
* `prob.py` 

### ソースコードの確認
`prob.py` を確認するとRSA暗号の鍵生成において `p = getPrime(512)` に対して `q = getPrime(32)` と設定されています．暗号文 `c` は標準的なRSA暗号の暗号化処理により生成されています．

## 3. 脆弱性と攻略方針

### 脆弱性
RSA暗号では通常 `p` と `q` は同程度の大きさの素数を使用しますが，本問題では `q` が32ビットと極めて小さいです．片方の素因数が小さすぎるためNの素因数分解が容易になるという脆弱性が存在します．

### 攻略方針：
`sympy` ライブラリの `pollard_rho` 関数を用いることでNから素因数 `q` を一瞬で抽出します．抽出に成功すると `p = n // q` を求められるため $\phi(n) = (p - 1)(q - 1)$ を計算できます．$\phi(n)$ が判明すると秘密鍵 `d` を導出できるため暗号文 `c` を復号してフラグを得ます．

## 4. エクスプロイトコード

```python
from Crypto.Util.number import long_to_bytes
from sympy.ntheory import pollard_rho

n = 28576441902831457590427748668696557474043630372677632241672926869035931925834848020457746529998588578832822195110996090327253913633985879817257290686867076047319213
e = 65537
c = 20416582581078679030890390494251758803999983698670759422923367333586408291714658619559562503769379652576538080202274167806350679198100959619974180892435535730583248

q = pollard_rho(n)

if q:
    p = n // q
    phi = (p - 1) * (q - 1)
    
    d = pow(e, -1, phi)
    m = pow(c, d, n)
    
    print(long_to_bytes(m).decode())
else:
    print("Failed to find q")
```
