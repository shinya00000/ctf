# Vending Machine writeup
<https://alpacahack.com/daily/challenges/vending-machine>

## 1. 概要

本問題は提供されたPythonスクリプトによる自動販売機プログラムの脆弱性を突いてフラグを獲得する問題です．

## 2. 問題の分析

### ファイル構成

* `server.py` のみ

### ソースコードの確認

`server.py` の中身を確認すると `VendingMachine` クラスが実装されており，`stock` 変数に各商品の在庫が文字列として定義されています．在庫の文字列は `a` から `e` の商品文字が連続しており，文字列の最後にフラグを表す `f` が1つだけ配置されています．`buy` メソッドではユーザーが指定した文字 `mark` を受け取りますが，ここで許可されているのは `a`，`b`，`c`，`d`，`e` のみです．また `buy` メソッド内において `find` メソッドを用いて `stock` から商品の位置 `loc` を検索し `pop(loc)` によって対象の商品を取り出しています．

## 3. 脆弱性と攻略方針

### 脆弱性

`buy` メソッドにおける `mark` の入力チェックには `f` が含まれていないため直接フラグを購入することはできません．しかし商品の在庫管理においてPythonの文字列検索メソッド `find` とリストの要素削除メソッド `pop` が組み合わせて使用されている点に脆弱性が存在します．`find` メソッドは検索対象の文字列が存在しない場合に `-1` を返す仕様となっています．そしてPythonのリストに対して `pop(-1)` を実行するとリストの末尾の要素が取り出されます．

### 攻略方針：

初期状態の `stock` の末尾にはフラグを示す `f` が配置されています．したがって特定の商品をすべて買い占めて在庫を空にすることで `find` メソッドに `-1` を返させることができれば末尾の `f` を `pop` することが可能となります．具体的には `a` を30回連続で購入することで `stock` から `a` を完全に枯渇させます．その後さらに `a` を購入しようとすると `find('a')` が `-1` を返すため `pop(-1)` が実行されてフラグを獲得できます．

## 4. エクスプロイトコード

Pwntoolsを使用したエクスプロイトコードを以下に示します．

```python
from pwn import *

io = remote('34.170.146.252', 30573)

io.recvuntil(b'your choice> ')

for i in range(30):
    io.sendline(b'a')
    io.recvuntil(b'your choice> ')

io.sendline(b'a')

output = io.recvall().decode()
print(output)

```
